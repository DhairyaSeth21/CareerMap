## System Overview

**CareerMap** is a full-stack web application that helps users navigate their career development through:
- **Personalized Career Paths**: Choose from 35 different tech career roles
- **AI-Powered Assessments**: Take quizzes generated by OpenAI to prove your skills
- **Dynamic Skill Tracking**: Real-time tracking of your proficiency in required skills
- **Next Action Recommendations**: Get personalized suggestions for what to do next
- **Role Readiness Score**: See exactly how prepared you are for your target role

### Technology Stack
- **Frontend**: Next.js 16, React, TypeScript, TailwindCSS
- **Backend**: Spring Boot (Java), JPA/Hibernate
- **Database**: MySQL
- **AI Integration**: OpenAI GPT API for quiz generation
- **Architecture**: RESTful API with CORS support

## Quick Start Guide

### Prerequisites
- Java 17+ (for backend)
- Node.js 18+ (for frontend)
- MySQL 8+ (for database)
- OpenAI API Key

### Step 1: Database Setup

```bash
# Start MySQL
mysql -u root -p

# Create database
CREATE DATABASE careermap;
USE careermap;
```

### Step 2: Start Backend

```bash
cd backend

# Option A: Use startup script (recommended)
chmod +x start-backend.sh
./start-backend.sh

# Option B: Manual startup
export DB_PASSWORD=your_db_password
export OPENAI_API_KEY=your_openai_api_key
./gradlew bootRun
```

**Backend will start on**: `http://localhost:8080`

### Step 3: Start Frontend

```bash
cd careermap-ui
npm install
npm run dev
```

**Frontend will start on**: `http://localhost:3000` (or 3001/3002 if 3000 is busy)

### Step 4: Test the Application

1. **Navigate to**: `http://localhost:3000/signup`
2. **Create an account**: Enter name, email, password
3. **Choose a role**: Select from 35 available career roles
4. **View your path**: See skill breakdown and readiness score
5. **Take assessments**: Click "Start Assessment" to prove your skills
6. **Track progress**: Watch your role readiness improve as you complete quizzes

## Features

### 1. User Authentication
- **Registration**: Create account with name, email, password
- **Login**: Secure authentication with session management
- **Persistent Sessions**: Stay logged in across browser sessions

### 2. Career Path Selection
- **35 Career Roles** across 5 categories:
  - Software Development (10 roles)
  - Infrastructure & Cloud (8 roles)
  - Data & AI (7 roles)
  - Security (4 roles)
  - Quality & Testing (2 roles)
  - Leadership (4 roles)
- **Dynamic Role Switching**: Change your target role anytime
- **Automatic Recalculation**: Path updates when you switch roles

### 3. Skill Tracking
- **Real-time Proficiency**: Track your current skill level vs. required
- **Gap Analysis**: See exactly what skills need improvement
- **Weighted Scoring**: Important skills contribute more to readiness
- **Visual Progress**: Color-coded skill bars (green/yellow/blue)

### 4. AI-Powered Assessments
- **OpenAI-Generated Quizzes**: Real, relevant questions for each skill
- **4 Difficulty Levels**: Beginner, Intermediate, Advanced, Expert
- **10 Questions per Quiz**: Focused, comprehensive assessments
- **Instant Feedback**: See results immediately after submission
- **Proficiency Awards**: Earn skill points based on quiz performance

### 5. Next Action Recommendations
- **Smart Prioritization**: Algorithm determines highest-impact next steps
- **Two Action Types**:
  - `ASSESS_SKILL`: Take a quiz to prove your knowledge
  - `FIND_OPPORTUNITY`: Apply your skills in real projects
- **One-Click Navigation**: Jump directly to assessments from suggestions

### 6. Role Readiness Score
- **0-10 Scale**: Clear metric of how prepared you are
- **Weighted Calculation**: Based on skill importance to the role
- **Real-time Updates**: Recalculates after every quiz

**Description**: Create a new user account

**Request Body**:
```json
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "securepass123"
```

**Response**:

## Database Schema

### Tables

#### `users`
```sql
CREATE TABLE users (
  user_id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL,
  primary_role_id INT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `roles`
```sql
CREATE TABLE roles (
  role_id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) UNIQUE NOT NULL,
  category VARCHAR(255) NOT NULL,
  description TEXT,
  icon VARCHAR(10),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `skills`
```sql
CREATE TABLE skills (
  skill_id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) UNIQUE NOT NULL,
  category VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### `role_skill_requirements`
```sql
CREATE TABLE role_skill_requirements (
  requirement_id INT AUTO_INCREMENT PRIMARY KEY,
  role_id INT NOT NULL,
  skill_id INT NOT NULL,
  required_level DECIMAL(3,1) DEFAULT 5.0,
  weight DECIMAL(3,2) DEFAULT 0.10,
  FOREIGN KEY (role_id) REFERENCES roles(role_id),
  FOREIGN KEY (skill_id) REFERENCES skills(skill_id)
);
```

#### `proficiencies`
```sql
CREATE TABLE proficiencies (
  proficiency_id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  skill_id INT NOT NULL,
  level DECIMAL(3,1) DEFAULT 0.0,
  last_assessed TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id),
  FOREIGN KEY (skill_id) REFERENCES skills(skill_id),
  UNIQUE KEY unique_user_skill (user_id, skill_id)
);
```

#### `quizzes`
```sql
CREATE TABLE quizzes (
  quiz_id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  skill_id INT NOT NULL,
  difficulty VARCHAR(50) NOT NULL,
  score DECIMAL(5,2),
  time_taken INT,
  proficiency_awarded DECIMAL(3,1),
  submitted_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(user_id),
  FOREIGN KEY (skill_id) REFERENCES skills(skill_id)
);
```

#### `quiz_questions`
```sql
CREATE TABLE quiz_questions (
  question_id INT AUTO_INCREMENT PRIMARY KEY,
  quiz_id INT NOT NULL,
  question_number INT NOT NULL,
  question_text TEXT NOT NULL,
  option_a TEXT NOT NULL,
  option_b TEXT NOT NULL,
  option_c TEXT NOT NULL,
  option_d TEXT NOT NULL,
  correct_answer CHAR(1) NOT NULL,
  user_answer CHAR(1),
  FOREIGN KEY (quiz_id) REFERENCES quizzes(quiz_id)
);
```

### Entity Relationships

```
users (1) -------- (many) proficiencies
users (1) -------- (1) roles
users (1) -------- (many) quizzes

roles (1) -------- (many) role_skill_requirements
skills (1) -------- (many) role_skill_requirements
skills (1) -------- (many) proficiencies
skills (1) -------- (many) quizzes

quizzes (1) -------- (many) quiz_questions
```

## Troubleshooting

### Common Issues

#### 1. CORS Errors
**Error**: `Access to fetch at 'http://localhost:8080/...' from origin 'http://localhost:3000' has been blocked by CORS policy`

**Solution**:
- Verify backend is running on port 8080
- Check that `WebConfig.java` includes your frontend port
- If frontend is on a different port (3001, 3002), update CORS configuration
- Restart backend after CORS changes

#### 2. Database Connection Failed
**Error**: `Could not open connection to database`

**Solution**:
```bash
# Verify MySQL is running
mysql -u root -p

# Check database exists
SHOW DATABASES;

# Create if missing
CREATE DATABASE careermap;

# Verify credentials in application.properties
spring.datasource.username=root
spring.datasource.password=${DB_PASSWORD}
```

#### 3. OpenAI API Errors
**Error**: `Quiz generation failed` or `401 Unauthorized`

**Solution**:
- Verify OpenAI API key is set: `echo $OPENAI_API_KEY`
- Check key is valid and has credits
- Update key in `start-backend.sh` or export manually
- Restart backend after updating key

#### 4. Frontend Build Errors
**Error**: `Parsing ecmascript source code failed`

**Solution**:
```bash
# Clean build artifacts
cd careermap-ui
rm -rf .next node_modules

# Reinstall dependencies
npm install

# Try building again
npm run build
```

#### 5. Port Already in Use
**Error**: `Port 8080 is already in use` or `Port 3000 is already in use`

**Solution**:
```bash
# Find process using port
lsof -i :8080
lsof -i :3000

# Kill process
kill -9 <PID>

# Or use different port for frontend
# Next.js will auto-assign 3001, 3002 if 3000 is busy
```

#### 6. Quiz Questions Are Placeholders
**Problem**: Quiz shows "Sample question" text instead of real questions

**Root Cause**: OpenAI API key not configured or invalid

**Solution**:
1. Verify environment variable is set:
   ```bash
   echo $OPENAI_API_KEY
   ```
2. If empty, export it:
   ```bash
   export OPENAI_API_KEY=your-key-here
   ```
3. Restart backend:
   ```bash
   ./gradlew bootRun
   ```

#### 7. Skill Assessment Navigation Not Working
**Problem**: Clicking "Start Assessment" doesn't pre-select skill

**Root Cause**: Frontend not passing skillId URL parameter

**Solution**:
- Verify button uses: `router.push(\`/assessment?skillId=\${action.skillId}\`)`
- Check browser URL shows: `http://localhost:3000/assessment?skillId=1`
- Clear browser cache and reload

#### 8. Role Readiness Not Updating
**Problem**: Readiness score stays at 0.0 after completing quiz

**Root Cause**: Proficiency not being saved to database

**Solution**:
1. Check backend logs for errors
2. Verify quiz submission returns proficiencyAwarded
3. Reload path page to see updated score
4. Check database:
   ```sql
   SELECT * FROM proficiencies WHERE user_id = 1;
   ```

### Key Algorithms

#### Role Readiness Calculation
```
readiness = Î£(skill_proficiency * skill_weight) / Î£(skill_weight)

Where:
- skill_proficiency = min(user_level / required_level * 10, 10)
- skill_weight = importance of skill to role (0.0 - 1.0)
- Result is 0.0 - 10.0 scale
```

#### Quiz Proficiency Award
```
proficiency = base_score * difficulty_multiplier * (1 - time_penalty)

Where:
- base_score = (correct_count / total_questions) * 10
- difficulty_multiplier:
  - Beginner: 0.6
  - Intermediate: 0.8
  - Advanced: 1.0
  - Expert: 1.2
- time_penalty = min((time_taken - expected_time) / expected_time * 0.1, 0.2)
```

#### Next Action Prioritization
```
1. Find skills with gap > 0
2. Sort by: gap * weight (descending)
3. If user has assessed < 3 skills: recommend ASSESS_SKILL
4. Otherwise: alternate between ASSESS_SKILL and FIND_OPPORTUNITY
5. Return top 3-5 actions
```
